3. Getting started
==================

1. Wait for the MediKeep pod to become ready:

   kubectl wait --for=condition=Ready pod -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }},app.kubernetes.io/name={{ include "medikeep.name" . }} --timeout=5m

2. Port-forward the application service:

   kubectl port-forward svc/{{ include "medikeep.fullname" . }} 8000:{{ .Values.service.port }} -n {{ .Release.Namespace }}

3. Open the MediKeep UI:

   http://127.0.0.1:8000

4. Log in with the default credentials and change them immediately.

7. Troubleshooting
==================

Inspect pods and events:

  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }} -o wide
  kubectl describe pod -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }},app.kubernetes.io/name={{ include "medikeep.name" . }}
  kubectl get events -n {{ .Release.Namespace }} --sort-by=.lastTimestamp

Inspect MediKeep logs:

  kubectl logs deploy/{{ include "medikeep.fullname" . }} -n {{ .Release.Namespace }} --tail=200
  kubectl logs deploy/{{ include "medikeep.fullname" . }} -n {{ .Release.Namespace }} --previous --tail=200

Common checks:

- If the pod stays in CrashLoopBackOff, check database connectivity and credentials.
- If migrations fail, verify database host, port, user, and password configuration.
- If uploads are lost after restart, verify persistence.uploads.enabled is true and the PVC is bound.
- If the UI is unreachable, verify ingress/Gateway configuration and DNS.
