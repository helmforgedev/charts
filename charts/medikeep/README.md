# MediKeep Helm Chart

Deploy [MediKeep](https://github.com/afairgiant/MediKeep), a self-hosted personal medical records application built with React and FastAPI.

This chart packages the official `ghcr.io/afairgiant/medikeep:v0.67.0` image with PostgreSQL, persistent uploads and backups,
Kubernetes Ingress, Gateway API, NetworkPolicy, and External Secrets Operator integration.

## Install

```bash
helm repo add helmforge https://repo.helmforge.dev
helm install medikeep helmforge/medikeep
```

MediKeep exposes HTTP on port `8000`. For local access:

```bash
kubectl port-forward svc/medikeep 8000:8000
```

## Production Notes

MediKeep stores personal health data. Treat the database, uploads PVC, backups PVC, logs, and Kubernetes Secrets as sensitive.

On fresh installs, upstream creates an `admin` user with the default password `admin123` unless `ADMIN_DEFAULT_PASSWORD` is set before first startup.
For production, provide a Secret through `secrets.existingSecret` or set `secrets.adminDefaultPassword` during the first install,
then rotate the password in the UI.

```yaml
secrets:
  existingSecret: medikeep-runtime
  secretKeyKey: secret-key
  adminDefaultPasswordKey: admin-password
```

The Secret must contain:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: medikeep-runtime
type: Opaque
stringData:
  secret-key: replace-with-a-long-random-secret
  admin-password: replace-with-a-strong-initial-password
```

## PostgreSQL

The chart enables the HelmForge PostgreSQL subchart by default:

```yaml
postgresql:
  enabled: true
  auth:
    database: medical_records
    username: medapp
```

To use an external PostgreSQL instance:

```yaml
postgresql:
  enabled: false

database:
  external:
    host: postgres.example.com
    name: medical_records
    username: medapp
    existingSecret: medikeep-db
    existingSecretPasswordKey: password
```

## Persistence

MediKeep writes to three paths:

- `/app/uploads` for uploaded lab files, patient photos, and record attachments.
- `/app/backups` for backups generated by the app dashboard or CLI.
- `/app/logs` for application logs.

Uploads and backups are persisted by default. Logs use `emptyDir` by default to avoid retaining sensitive data unless explicitly requested.

```yaml
persistence:
  uploads:
    size: 20Gi
  backups:
    size: 20Gi
  logs:
    enabled: true
    size: 2Gi
```

## Exposure

Ingress:

```yaml
ingress:
  enabled: true
  ingressClassName: traefik
  hosts:
    - host: medikeep.example.com
      paths:
        - path: /
          pathType: Prefix
```

Gateway API:

```yaml
gatewayAPI:
  enabled: true
  httpRoutes:
    - parentRefs:
        - name: public
          namespace: gateway-system
      hostnames:
        - medikeep.example.com
```
