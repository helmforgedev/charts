Initial admin password
======================

Gophish writes the generated initial admin password to startup logs on first boot.
Retrieve it only from trusted operator workstations and change it immediately after login:

- kubectl logs deploy/{{ include "gophish.fullname" . }} -n {{ .Release.Namespace }} | grep "Please login with the username admin"

Troubleshooting
===============

Inspect workload status and logs:
- kubectl describe pod -l app.kubernetes.io/instance={{ .Release.Name }} -n {{ .Release.Namespace }}
- kubectl logs -l app.kubernetes.io/instance={{ .Release.Name }} -n {{ .Release.Namespace }}

Common checks:
- admin UI not reachable: confirm admin service port-forward or admin ingress host/TLS
- phishing endpoint not reachable: confirm phish service, phish ingress, and DNS
- initial password not found: inspect first startup logs before log retention expires
- SQLite issues: confirm PVC is bound and the pod can write {{ include "gophish.sqliteDirectory" . }}
- MySQL issues: confirm DSN Secret, service reachability, and Gophish-compatible MySQL sql_mode
- backup issues: inspect the backup Job logs and the S3 credential Secret
