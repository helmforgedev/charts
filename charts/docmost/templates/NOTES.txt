6. Validation Commands
======================
  # Check pods are ready
  kubectl get pods -n {{ $namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

  # Check services, PVCs, and optional routes
  kubectl get svc,pvc -n {{ $namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}
  kubectl get ingress,httproute -n {{ $namespace }} -l app.kubernetes.io/instance={{ .Release.Name }} 2>/dev/null || true

  # Check Docmost logs
  kubectl logs -n {{ $namespace }} deploy/{{ include "docmost.fullname" . }} -c docmost --tail=100

  # Check dependency init containers when startup is blocked
  kubectl logs -n {{ $namespace }} deploy/{{ include "docmost.fullname" . }} -c wait-for-postgresql --tail=100
  kubectl logs -n {{ $namespace }} deploy/{{ include "docmost.fullname" . }} -c wait-for-redis --tail=100

  # Test HTTP health from inside the pod
  kubectl exec -n {{ $namespace }} deploy/{{ include "docmost.fullname" . }} -- \
    sh -c "node -e \"fetch('http://127.0.0.1:3000/api/health').then(r=>{console.log(r.status);return r.text()}).then(t=>console.log(t)).catch(e=>{console.error(e);process.exit(1)})\""

7. Troubleshooting
==================
  Pod not ready:
    -> Check init container logs for PostgreSQL and Redis connectivity.
    -> Verify database.mode, database.external.*, redis.enabled, and redis.external.*.

  Login or session errors after upgrade:
    -> Ensure docmost.appSecret did not change.
    -> Prefer an existing Secret for production-grade secret management.

  Uploads disappear after pod replacement:
    -> Use storage.local.enabled=true with a PVC or configure storage.mode=s3.
    -> For more than one replica, prefer S3-compatible storage.

  Gateway route missing after upgrade:
    -> Existing gatewayAPI.* values still render HTTPRoute for compatibility.
    -> For new values, migrate to gateway.enabled, gateway.parentRefs, and gateway.hostnames.

  Database extension or search issues:
    -> Bundled PostgreSQL initializes unaccent and pg_trgm on fresh databases.
    -> Existing external databases should have those extensions enabled before use.

  Backup job failed:
    -> Check backup.s3.endpoint, bucket, and credentials.
    -> Inspect job logs:
       kubectl logs -n {{ $namespace }} job/<backup-job-name>
