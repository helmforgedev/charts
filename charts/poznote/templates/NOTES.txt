3. Getting started
==================

1. Wait for the Poznote pod to become ready:

   kubectl wait --for=condition=Ready pod -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }},app.kubernetes.io/name={{ include "poznote.name" . }} --timeout=5m

2. Port-forward the application service:

   kubectl port-forward svc/{{ include "poznote.fullname" . }} 8080:{{ .Values.service.port }} -n {{ .Release.Namespace }}

3. Open the Poznote UI:

   http://127.0.0.1:8080

4. Log in with the default credentials and change them immediately.

7. Troubleshooting
==================

Inspect pods and events:

  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }} -o wide
  kubectl describe pod -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }},app.kubernetes.io/name={{ include "poznote.name" . }}
  kubectl get events -n {{ .Release.Namespace }} --sort-by=.lastTimestamp

Inspect Poznote logs:

  kubectl logs deploy/{{ include "poznote.fullname" . }} -n {{ .Release.Namespace }} --tail=200
  kubectl logs deploy/{{ include "poznote.fullname" . }} -n {{ .Release.Namespace }} --previous --tail=200

Common checks:

- If the pod stays in CrashLoopBackOff, check logs for permission errors on the data volume.
- If OIDC login fails, verify the OIDC credentials Secret exists and contains the correct keys.
- If the pod starts but the UI is unreachable, verify ingress/Gateway configuration and DNS.
- If data is lost after pod restart, verify persistence.data.enabled is true and the PVC is bound.
